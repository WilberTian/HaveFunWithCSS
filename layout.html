position: static

Normally, the position will be default as "static", which means the element is not positioned and stays where it normally would in the document flow.

With static, the top/right/bottom/left will not work

position: relative

When the position is set to relative, the element will still hold the original place in the document flow.

Then with top/right/bottom/left, the element can be moved to the relative place

position: absolute

When the position is set to absolute, the element is removed from the document flow.

This element will placed exactly with top/right/bottom/left based on the first positioned parent (if all the parents are not position, the absolute element will be placed based on body element).

position:relative + position:absolute

When the position is set to absolute, the element is removed from the document flow.

This element will placed exactly with top/right/bottom/left based on the first positioned parent (if all the parents are not position, the absolute element will be placed based on body element).

position:absolute will affect display

The absolute element will be removed from document flow, then the display of the element will be changed

position: fixed

When the position is set to absolute, the element is removed from the document flow.

This element will placed exactly with top/right/bottom/left based on browser window.

position:fixed will affect display

The fixed element will be removed from document flow, then the display of the element will be changed
